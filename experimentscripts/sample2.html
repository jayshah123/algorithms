PHP: Extract Outgoing URLs from a Web Page

Posted by Joys of Programming on in PHP, Web

In PHP, you can download a web page using file_get_contents or curl. Once you have downloaded a web page, you can process it.

We know that the tag structure of hyperlink is as follows

<a href="http://www.joysofprogramming.com">Joys of Programming</a>

Keeping this in mind, we write the following program

<?php

function extractElementsFromWebPage($webPage, $tagName) {
  //Creating a DOMDocument Object.
  $dom = new DOMDocument;

  //Parsing the HTML from the web page
  if ($dom->loadHTML($webPage)) {
    // Extracting the specified elements from the web page
    @$elements = $dom->getElementsByTagName($tagName);
    return $elements;
  }
  return FALSE;
}

function downloadURL($URL) {
  $webPage = file_get_contents ($URL);
  return $webPage;
}

$webPage = downloadURL("http://www.mozilla.org/");
if ($webPage ) {
  $URLs = extractElementsFromWebPage($webPage, 'a');
  if ($URLs) {
    foreach ($URLs as $URL){
      // Extracting the URLs
      echo $URL->getAttribute('href'), "\n";
    }
  }
  else {
    echo "Error in parsing the webPage\n";
  }
}
else {
  echo "Error in downloading the webPage\n";
}
?>

There are certain things that need to be understood:

Firstly we are using file_get_contents to download a web page. Then we use the DOMDocument class in PHP to parse the HTML page. Check the two functions

  1. downloadURL
  2. extractElementsFromWebPage

downloadURL uses file_get_contents to download the web page and extractElementsFromWebPage uses the DOMDocument class. The function loadHTML is used to parse the HTML page and getElementsByTagName to extract the specified elements. In our case, we want to extract the HTML tag element a.

Google Chrome: Unable to Open in Mac

Posted by Joys of Programming on in Google Chrome, Mac

There can be many reasons why you can’t open Google Chrome in Mac. To diagnose this issue, try typing the following in the Terminal and run Chrome from the terminal

$ /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome
[......:ERROR:process_singleton_mac.cc(102)] Unable to obtain profile lock.

If you get the following error, you must remove the following file

$ rm ~/Library/Application\ Support/Google/Chrome/SingletonLock

And now try opening Google Chrome.

Thunderbird: A copy of Thunderbird is already open. Only one copy of Thunderbird can be open at a time.

Posted by Joys of Programming on in Thunderbird

If you are trying to open Thunderbird in Mac and get the following alert

A copy of Thunderbird is already open. Only one copy of Thunderbird can be open at a time.

There are two possibilities:

Make sure that you don’t have already opened Thunderbird. How can you make sure that Thunderbird is running or not. Go to the Terminal

$ ps aux | grep -i thunderbird
abc    1412   1,3  0,0  2435116    508 s000  R+    3:36     0:00.01 grep -i thunderbird
abc    1297   0,0  1,8  3533484 153400   ??  S     3:29     0:04.62 /Volumes/Thunderbird/Thunderbird.app/Contents/MacOS/thunderbird-bin -psn_0_803012


It may be in the process of opening and you clicked on the Thunderbird icon twice. Simply press Ok and wait for the Thunderbird to open.

You are sure that the Thunderbird is not running, which will give an output like this

$ ps aux | grep -i thunderbird
abc    1412   1,3  0,0  2435116    508 s000  R+    3:36     0:00.01 grep -i thunderbird

This may be because you need to remove a lock file from your system. In Mac, this file is found in

$ ls ~/Library/Thunderbird/Profiles/
1stutbrt.default

Now that you have got your profile name, you need to remove the file named .parentlock

$ rm ~/Library/Thunderbird/Profiles/1stutbrt.default/.parentlock

Now try opening Thunderbird.

Google Chrome: Unable to obtain profile lock

Posted by Joys of Programming on in Google Chrome

If you have ever come across this issue while working with Google Chrome in Mac

[436:2307:2779210513702:ERROR:process_singleton_mac.cc(102)] Unable to obtain profile lock.

It is because the Singleton lock has to be deleted. This problem occurs because of premature shutdown of the machine while the Google Chrome was running

In Mac OS, remove the following file as shown below

$ rm ~/Library/Application\ Support/Google/Chrome/SingletonLock

And now try opening Google Chrome.
